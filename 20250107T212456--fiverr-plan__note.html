  • Credibility
    If you are wondering about how you will get more data for your buck, the answer is, a "Decaying formula".
    \[ $50 + \frac{NumberOfRows}{\ln({NumberOfRows})^3} * e^{SecurityLevel}\]
    • The first part is a flat fee
    • The second part is \(\frac{NumberOfRows}{\ln({NumberOfRows})^3}\) a fraction that lower the price as the number of rows increase.
      Compared to other with a linear pricing model.
    • The third part is \(e^{SecurityLevel}\) a representation of data acquiring difficulty.
      There are three levels:
      • The security level 1: is when there is little to no hurdles (Pacifist developers, Hobbits)
      • The security level 2: is when there is one or more restrictions, like Cloudflare, requiring GUI, or account login to name a few. (Military base level, Nazgul)
      • The security level 3: is when the web developers want to keep their precious data to themselves…(Gollum my precious)
  • Call to action 2
